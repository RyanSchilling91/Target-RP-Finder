# TRINITY_SCHEMA.md

## Purpose
This document is the authoritative schema definition for Trinity's 
SQLite database. db.py must create these tables exactly as specified 
on first launch. Trinity does not interpret the meaning of any values 
— it stores and retrieves them as instructed by the caller.

---

## Table: runs

| Column       | Type    | Constraints           | Notes                          |
|--------------|---------|-----------------------|--------------------------------|
| run_id       | TEXT    | PRIMARY KEY           | UUID generated by Trinity      |
| lims_id      | TEXT    | NOT NULL, UNIQUE      | Supplied by caller             |
| instrument   | TEXT    | NOT NULL              | Supplied by caller             |
| created_by   | TEXT    | NOT NULL              | os.getlogin() at creation time |
| created_at   | TEXT    | NOT NULL              | ISO 8601 UTC timestamp         |
| status       | TEXT    | NOT NULL              | Default: 'draft'               |
| current_rev  | TEXT    | FK → revisions.revision_id | Updated on each save      |

---

## Table: revisions

| Column       | Type    | Constraints           | Notes                          |
|--------------|---------|-----------------------|--------------------------------|
| revision_id  | TEXT    | PRIMARY KEY           | UUID generated by Trinity      |
| run_id       | TEXT    | NOT NULL, FK → runs   |                                |
| revision_num | INTEGER | NOT NULL              | Starts at 1                    |
| created_by   | TEXT    | NOT NULL              | Actor who created the revision |
| created_at   | TEXT    | NOT NULL              | ISO 8601 UTC timestamp         |
| state_json   | TEXT    | NOT NULL              | JSON string, default '{}'      |

---

## Table: locks

| Column       | Type    | Constraints                | Notes                          |
|--------------|---------|----------------------------|--------------------------------|
| lock_id      | TEXT    | PRIMARY KEY                | UUID generated by Trinity      |
| run_id       | TEXT    | NOT NULL, UNIQUE, FK → runs| One lock per run maximum       |
| owner        | TEXT    | NOT NULL                   | os.getlogin() of lock holder   |
| acquired_at  | TEXT    | NOT NULL                   | ISO 8601 UTC timestamp         |
| heartbeat_at | TEXT    | NOT NULL                   | Updated every 60 seconds       |
| workstation  | TEXT    | NOT NULL                   | os.environ COMPUTERNAME        |

---

## Table: audit_events

| Column      | Type  | Constraints         | Notes                            |
|-------------|-------|---------------------|----------------------------------|
| event_id    | TEXT  | PRIMARY KEY         | UUID generated by Trinity        |
| run_id      | TEXT  | NOT NULL, FK → runs |                                  |
| revision_id | TEXT  | FK → revisions      | Nullable — not all events have one|
| actor       | TEXT  | NOT NULL            | os.getlogin() at event time      |
| event_type  | TEXT  | NOT NULL            | Constant string from audit_log.py|
| note        | TEXT  |                     | Human-readable description       |
| timestamp   | TEXT  | NOT NULL            | ISO 8601 UTC timestamp           |

Audit rows are append-only. No update or delete is ever 
performed on this table.

---

## Database settings

- WAL mode enabled on every connection: PRAGMA journal_mode=WAL
- Foreign keys enforced: PRAGMA foreign_keys=ON
- All timestamps stored as ISO 8601 UTC strings, never as integers

---

## Stale lock constants

- Stale threshold: 20 minutes
- Heartbeat interval: 60 seconds
- Both defined as constants in lock_manager.py
